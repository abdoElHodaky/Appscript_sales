# دليل المساهمة / Contributing Guide

## 🇦🇪 العربية

شكراً لاهتمامك بالمساهمة! 🎉

### كيفية المساهمة

1. **Fork** المستودع
2. أنشئ **branch** جديد: `git checkout -b feature/اسم-الميزة`
3. **Commit** التغييرات: `git commit -m "وصف التغيير"`
4. **Push** إلى الـ branch: `git push origin feature/اسم-الميزة`
5. افتح **Pull Request**

### معايير الكود

- استخدم `const` و `let` بدلاً من `var`
- أضف تعليقات JSDoc لكل دالة
- اختبر الكود قبل الرفع: `npm test`
- تأكد من عدم وجود secrets في الكود

### الإبلاغ عن الأخطاء

استخدم قالب [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md)

### طلب ميزات جديدة

استخدم قالب [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md)

## 🇺🇸 English

Thank you for your interest in contributing! 🎉

### How to Contribute

1. **Fork** the repository
2. Create a new **branch**: `git checkout -b feature/feature-name`
3. **Commit** changes: `git commit -m "Description of change"`
4. **Push** to branch: `git push origin feature/feature-name`
5. Open a **Pull Request**

### Code Standards

- Use `const` and `let` instead of `var`
- Add JSDoc comments for every function
- Test code before pushing: `npm test`
- Ensure no secrets in code

### Reporting Bugs

Use [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) template

### Requesting Features

Use [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md) template
